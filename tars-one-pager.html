Core Modules
Seven Specialized Agents, One Unified Platform
💬
Chat Handler
Natural language queries against CloudWatch logs. Two modes: Quick (real-time filter_log_events) and Deep (Logs Insights analytics). Powered by Claude 3.5 Sonnet synthesis.
🚨
Incident Handler
CloudWatch alarm auto-triage. When an alarm fires, TARS auto-investigates the relevant service logs, identifies the incident window, and returns a structured diagnosis.
🧠
Diagnosis Agent
AI-powered root cause analysis. The LLM reasons over correlated log evidence, identifies the failure chain, and generates a human-readable RCA with confidence indicators.
🔧
Remediation Handler
Auto-generates GitHub Pull Requests with fixes, creates issues, and tracks merge status. Human-in-the-loop approval gate before execution of any changes.
📚
Knowledge Base (RAG)
Upload runbooks, SOPs, and incident post-mortems. Titan V2 embeddings + cosine similarity retrieval. KB context is injected into every chat response — with source attribution.
🔗
Correlation Engine
Traces CORR-IDs across 8+ microservices. One query maps the full request journey: API Gateway → Auth → Orders → Payments → Inventory — identifying where the chain broke.
💾
Session Manager
Persistent conversation history stored in DynamoDB. Teams revisit past investigations, build on prior context, and maintain institutional memory across incidents.
📈
Multi-Mode Search
Adaptive query routing: Quick mode for real-time triage, Deep mode for time-range analytics. Automatically selects the right CloudWatch API based on query intent.
MCP Integrations
Plug Any Observability Platform via MCP
🧩
Model Context Protocol (MCP) is an open standard that lets observability platforms expose their data as tool-callable servers. TARS acts as an MCP host — each platform ships its own MCP server, and TARS plugs in automatically. No custom connectors. No per-platform code. Just register the MCP endpoint and TARS gains full query access to that platform's logs, metrics, and traces.

Datadog Roadmap
📦 datadog-mcp-server
Datadog's official MCP server exposes metrics, logs, APM traces, and monitors. TARS hooks in to query dashboards and alert history via natural language.
Dynatrace Roadmap
📦 dynatrace-mcp-server
Dynatrace MCP server surfaces Davis AI problems, USQL query results, and distributed traces. TARS injects findings into its diagnosis agent automatically.
Elasticsearch / OpenSearch Roadmap
📦 elasticsearch-mcp-server
Community MCP server for Elasticsearch / OpenSearch indices. TARS sends DSL queries via tool calls and retrieves structured log results for AI analysis.
Splunk Roadmap
📦 splunk-mcp-server
Splunk MCP server exposes SPL search, saved searches, and alert history as callable tools. TARS correlates Splunk data with KB context for enriched answers.
PagerDuty Roadmap
📦 pagerduty-mcp-server
PagerDuty MCP server provides incident history, on-call schedules, and alert routing rules. TARS links live log evidence directly to open incidents and escalation paths.
